What I learned:

  1. Github makes a distinction between a user and a project page. The user page is created by selecting a special repo name; the project page is created with a specially named branch. Both only require the presence of index.html in the root of the repo.
  2. For project pages, if you don't select gh-pages as the default branch then you'll have to keep merging into
  3. It's possible to use a custom domain name.
